Introducción al Proyecto

Este proyecto académico implementa disparadores (triggers) en SQL Server para automatizar procesos, reforzar la integridad de datos y proporcionar mecanismos de auditoría en las bases de datos Northwind y Pubs.

Objetivo General

Desarrollar e implementar un conjunto de triggers que automaticen procesos, controlen reglas de negocio y auditen eventos en las bases de datos.

Base de Datos Northwind

Base de datos de ejemplo de Microsoft que simula una empresa de ventas con productos, clientes, pedidos y empleados.

Base de Datos Pubs

Base de datos de ejemplo que simula una editorial de libros con autores, títulos, publicaciones y ventas.

1. Actualización Automática de Stock

AFTER INSERT

Tabla: Order Details

Propósito: Descuenta automáticamente las unidades compradas del inventario cuando se registra un nuevo detalle de orden.

Lógica: Después de insertar un registro en Order Details, se actualiza el stock del producto correspondiente.

CREATE TRIGGER trg_UpdateStock
ON [Order Details]
AFTER INSERT
AS
BEGIN
    UPDATE Products
    SET UnitsInStock = UnitsInStock - i.Quantity
    FROM Products p
    INNER JOIN inserted i ON p.ProductID = i.ProductID;
END;

2. Prevención de Eliminación de Órdenes

INSTEAD OF DELETE

Tabla: Orders

Propósito: Evita la eliminación accidental de órdenes para preservar la integridad histórica.

Lógica: Intercepta intentos de DELETE y muestra un mensaje en lugar de ejecutar la eliminación.

CREATE TRIGGER trg_PreventDeleteOrder
ON Orders
INSTEAD OF DELETE
AS
BEGIN
    PRINT 'No está permitido eliminar órdenes. Contacte al administrador.';
END;

3. Auditoría de Cambios de Precios

AFTER UPDATE

Tabla: Products

Propósito: Registra cambios de precio en una tabla de auditoría para rastreo.

Lógica: Detecta cambios en UnitPrice y los registra en Products_Audit con fecha y valores antiguo/nuevo.

CREATE TRIGGER trg_AuditPriceChange
ON Products
AFTER UPDATE
AS
BEGIN
    IF UPDATE(UnitPrice)
    BEGIN
        INSERT INTO Products_Audit (ProductID, OldPrice, NewPrice, ChangeDate)
        SELECT d.ProductID, d.UnitPrice, i.UnitPrice, GETDATE()
        FROM deleted d
        JOIN inserted i ON d.ProductID = i.ProductID;
    END
END;

4. Validación de Fecha de Envío

AFTER INSERT, UPDATE

Tabla: Orders

Propósito: Garantiza que la fecha de envío nunca sea anterior a la fecha de orden.

Lógica: Verifica la consistencia de fechas y revierte la transacción si hay inconsistencia.

CREATE TRIGGER trg_ValidateDates
ON Orders
AFTER INSERT, UPDATE
AS
BEGIN
    IF EXISTS (SELECT * FROM inserted WHERE ShippedDate < OrderDate)
    BEGIN
        RAISERROR('La fecha de envío no puede ser anterior a la fecha de la orden.', 16, 1);
        ROLLBACK TRANSACTION;
    END
END;

5. Estándar en Nombres de Clientes

AFTER INSERT, UPDATE

Tabla: Customers

Propósito: Mantiene uniformidad convirtiendo CompanyName a mayúsculas.

Lógica: Actualiza automáticamente el nombre de la empresa a mayúsculas después de insertar o actualizar.

CREATE TRIGGER trg_UpperCaseCustomer
ON Customers
AFTER INSERT, UPDATE
AS
BEGIN
    UPDATE Customers
    SET CompanyName = UPPER(i.CompanyName)
    FROM Customers c
    JOIN inserted i ON c.CustomerID = i.CustomerID;
END;

P1. Restricción de Regalías

AFTER UPDATE

Tabla: titles

Propósito: Evita que el porcentaje de regalías sea disminuido, protegiendo acuerdos contractuales.

Lógica: Compara valores antiguos y nuevos de royalty, revierte si hay disminución.

CREATE TRIGGER trg_CheckRoyalty
ON titles
AFTER UPDATE
AS
BEGIN
    IF EXISTS (
        SELECT * 
        FROM inserted i 
        JOIN deleted d ON i.title_id = d.title_id
        WHERE i.royalty < d.royalty
    )
    BEGIN
        RAISERROR('No se puede reducir el porcentaje de regalías.', 16, 1);
        ROLLBACK TRANSACTION;
    END
END;

P2. Auditoría de Empleados Nuevos

AFTER INSERT

Tabla: employee

Propósito: Registra automáticamente cada nuevo empleado en una tabla de auditoría.

Lógica: Inserta un registro en Employee_Log con la acción y fecha de ingreso.

CREATE TRIGGER trg_NewEmployeeLog
ON employee
AFTER INSERT
AS
BEGIN
    INSERT INTO Employee_Log (EmpID, Action, Date)
    SELECT emp_id, 'Nuevo Ingreso', GETDATE()
    FROM inserted;
END;

P3. Eliminación en Cascada de Ventas

INSTEAD OF DELETE

Tabla: titles

Propósito: Elimina ventas asociadas antes de eliminar un título, evitando errores por llaves foráneas.

Lógica: Primero elimina registros relacionados en sales, luego elimina el título.

CREATE TRIGGER trg_DeleteTitleSales
ON titles
INSTEAD OF DELETE
AS
BEGIN
    DELETE FROM sales 
    WHERE title_id IN (SELECT title_id FROM deleted);

    DELETE FROM titles 
    WHERE title_id IN (SELECT title_id FROM deleted);
END;

P4. Alerta por Stock Mínimo

AFTER INSERT

Tabla: sales

Propósito: Genera alerta cuando el stock baja de 10 unidades después de una venta.

Lógica: Verifica el stock después de insertar una venta y muestra mensaje de alerta si es bajo.

CREATE TRIGGER trg_StockAlert
ON sales
AFTER INSERT
AS
BEGIN
    IF EXISTS (
        SELECT * 
        FROM titles t
        JOIN inserted i ON t.title_id = i.title_id
        WHERE t.stock < 10
    )
    BEGIN
        PRINT 'ALERTA: El stock ha caído por debajo del mínimo permitido.';
    END
END;

P5. Asignación Automática de Fecha de Contratación

INSTEAD OF INSERT

Tabla: employee

Propósito: Asigna fecha actual si un empleado es insertado sin fecha de contratación.

Lógica: Intercepta inserción y asigna GETDATE() si hire_date es NULL.

CREATE TRIGGER trg_DefaultHireDate
ON employee
INSTEAD OF INSERT
AS
BEGIN
    INSERT INTO employee (emp_id, fname, lname, hire_date)
    SELECT emp_id, fname, lname,
           ISNULL(hire_date, GETDATE())
    FROM inserted;
END;
